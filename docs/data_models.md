# Модели данных

## Перечисление `Gender`
Используется для указания пола пациента.
*Базовый тип:* `uint8_t`

|Значение|Описание|
|--------|--------|
|`Male`|Мужской|
|`Female`|Женский|

В JSON передаётся строкой: `"gender": "male"`.

---
## Перечисление `PatientStatus`
Используется для отображения текущего статуса оказания помощи пациенту.
*Базовый тип:* `uint8_t`

|Значение|Описание|
|--------|--------|
|`OnTheWay`|Находится в карете скорой помощи|
|`Waiting`|Находится в больнице, в очереди в операционную|
|`InSurgery`|Находится на операции|
|`IntensiveCare`|После операции переведён в реанимацию|
|`Died`|Умер|

В JSON передаётся строкой: `"status": "on_the_way"`.

---

## Структура `Patient`
Предоставляет информацию о пострадавшем, ожидающем операции.

Поля расположены по убыванию размера с учётом выравнивания.

|Поле|Тип|Обязательное|Описание|
|----|---|------------|--------|
|`timestamp_`|`std::chrono::system_clock::time_point`|Да|Время создания запроса (серверное)|
|`mask_`|`uint32_t`|Да|Битовая маска симптомов. Каждый бит соответствует определённому симптому (0 – отсутствует, 1 – присутствует)|
|`id_`|`uint32_t`|Да|Уникальный идентификатор пациента (автоинкрементный)|
|`priority_`|`uint8_t`|Да|Приоритет пациента (чем меньше число, тем выше приоритет). Вычисляется сервером на основе `mask`|
|`status_`|`PatientStatus`|Да|Статус пациента, подробнее ([см. выше](#перечисление-status))|
|`age_`|`std::optional<uint8_t>`|Нет|Возраст пациента (если предоставлен клиентом). Диапазон 0–255.|
|`sex_`|`std::optional<Gender>`|Нет|Пол пациента (если предоставлен клиентом)|


### Формат представления `timestamp` в `JSON`
Количество миллисекунд с `1970-01-01 UTC`. Например, `"timestamp": 1717588800000`

### JSON для запроса `POST /patients`
#### JSON, соответствующий запросу
```json
{
    "mask": 13,     // uint32_t
    "age": 42,      // uint8_t
    "sex": "male"   // std::string
}
```

#### JSON, соответствующий ответу
```json
{
    "id": 123,                  // uint32_t
    "estimated_wait_time": 15   // uint16_t - время в минутах
}
```

### JSON для запроса `GET /patients/{id}`

#### JSON, соответствующий запросу
Не используется, `ID` выделяется из самого HTTP-запроса

#### JSON, соответствующий ответу
```json
{
    "timestamp": 1717588800000,         // uint32_t
    "id": 123,                          // uint32_t
    "mask": 13,                         // uint32_t
    "priority": 4,                      // uint8_t
    "status": "on_the_way",             // std::string
    "age": 42,                          // uint8_t (опционально)
    "sex": "male"                       // std::string (опционально)
}
```

### JSON для ответа на запрос `GET /patients`
```json
    {
        "timestamp": 1717588800000, // uint32_t
        "id": 1,                    // uint32_t
        "mask": 13,                 // uint32_t
        "priority": 2,              // uint8_t
        "status": "waiting",        // std::string
        "age": 42,                  // uint8_t (опционально)
        "sex": "male"               // std::string (опционально)
        
    },
    {
        "id": 2,
        "mask": 7,
        "priority": 1,
        "timestamp": 1717588810000,
        "status": "in_surgery"
    }
```
